### البطاقة التالية المقترحة
- إشعارات الجناح (/wing/notifications)

### لماذا هذه البطاقة الآن؟
- غير مكتملة حاليًا وتكمّل دورة عمل مشرف الجناح مع تأثير مباشر.
- لا تتطلب تغييرات Backend كبيرة مبدئيًا؛ يمكن تقديم نسخة MVP واجهية والربط لاحقًا.

### نطاق نسخة MVP (واجهية فقط)
- شريط عنوان موحّد مع أيقونة واضحة.
- بطاقة أدوات (تاريخ من/إلى، صف مقيَّد بالجناح، حالة، بحث).
- جدول بتحديد متعدد + "تحديد الكل" للعناصر الظاهرة.
- إجراءات مجمّعة مع RBAC: إرسال / إلغاء / أرشفة.
- حفظ الفلاتر في الرابط (URL) وLocalStorage.
- حالات تحميل/خطأ/فراغ وaria-live لنتائج الأوامر.

### خطوات التنفيذ
1) مراجعة WingNotifications.vue وclient.ts وتحديد فجوات الاستدعاءات.
2) بناء الواجهة (Header + Tools + Table + Bulk actions) مع RBAC.
3) ربط فلاتر الصفوف بالجناح المختار (selectedWingId) للسوبر.
4) دعم إلغاء الطلب السابق (AbortController) وDebounce للبحث.
5) اختبار يدوي وتسليم Demo.

### بعد الإشعارات
- تقارير الجناح (/wing/reports) كـ MVP: قوالب CSV/Word للتقارير السريعة.
